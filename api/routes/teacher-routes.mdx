## Overview

Teacher routes are divided into two categories:
1. **Docente-Only Routes**: Exclusive to users with the `docente` role
2. **Shared Routes**: Available to both `docente` and `psicoorientador` roles

<Warning>
  All teacher routes require:
  - `PreventBackHistoryMiddleware` - Prevents browser back button caching
  - `auth` - User must be authenticated
  - Role-specific middleware (see each section)
</Warning>

---

## Middleware Reference

### RoleDocenteMiddleware

Restricts access to users with the `docente` role only.

**Behavior**:
- Checks if user is authenticated
- Verifies user has `docente` role
- Returns 403 Forbidden if role check fails

**Error Message**: "No tienes permiso para acceder a esta página."

**Source**: RoleDocenteMiddleware.php:13-21

---

### RolePsicoorientadorAndDocenteMiddleware

Allows access to users with either `docente` or `psicoorientador` role.

**Purpose**: For activities that both teachers and psychologists have in common.

**Behavior**:
- Checks if user is authenticated
- Verifies user has either `docente` or `psicoorientador` role
- Returns 403 Forbidden if role check fails

**Error Message**: "No tienes permiso para acceder a esta página."

**Source**: RolePsicoorientadorAndDocenteMiddleware.php:13-21

## Common Use Cases

### Teacher Workflow

1. **Create Referral**: Use `GET /create/referral` to access the form
2. **Submit Referral**: Send data via `POST /store/referral`
3. **View Remitted Students**: Check status with `GET /index/students/remitted`
4. **Add Observations**: Use `GET /addMinutes` to add notes

### Student Management

1. **Edit Student**: Use `GET /edit/student/{id}` to access the form
2. **Update Student**: Submit changes via `PUT /update/student/{id}`
