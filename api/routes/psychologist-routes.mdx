## Overview

Psychologist routes provide comprehensive student support management including referral processing, student categorization (Active, PIAR, DUA), report generation, and historical tracking.

<Warning>
  All psychologist routes require:
  - `PreventBackHistoryMiddleware` - Prevents browser back button caching
  - `auth` - User must be authenticated
  - `RolePsicoorientadorMiddleware` - User must have `psicoorientador` role
</Warning>

## Middleware Reference

### RolePsicoorientadorMiddleware

Restricts access to users with the `psicoorientador` role only.

**Behavior**:
- Checks if user is authenticated
- Verifies user has `psicoorientador` role

## Common Workflows

### Processing New Referrals

1. **View Referrals**: Use `GET /index/students/remitted/psico`
2. **Review Details**: Access `GET /details/referral/{id}`
3. **Update Status**: Submit changes via `PUT /edit/details/referral/{id}`
4. **Create Report**: Use `GET /report/student/{id}` then `POST /store/report/student/{id}`
5. **Assign to PIAR**: If needed, use `PUT /accept/student`

### Student Monitoring

1. **View Active Students**: Use `GET /psico/students/active`
2. **Check PIAR Students**: Access `GET /psico/students/piar`
3. **Review DUA Students**: Use `GET /psico/students/dua`
4. **View Student History**: Access `GET /student/history/{id}`

### Report Management

1. **View Current Report**: Use `GET /psico/student/{id}/report`
2. **Create New Report**: Access form at `GET /report/student/{id}`
3. **Submit Report**: Post data via `POST /store/report/student/{id}`
4. **Review Historical Reports**: Use `GET /history/details/report/{id}`
5. **Update Historical Data**: Submit via `PUT /edit/history/details/report/{id}`
