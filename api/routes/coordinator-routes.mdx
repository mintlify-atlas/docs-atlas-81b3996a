## Overview

Coordinator routes are restricted to users with the `coordinador` role. These routes manage users, groups, degrees, and subject areas within the system.

<Warning>
  All coordinator routes require:
  - `PreventBackHistoryMiddleware` - Prevents browser back button caching
  - `auth` - User must be authenticated
  - `RoleMiddleware` - User must have `coordinador` role
</Warning>

## Middleware Reference

### RoleMiddleware

Restricts access to users with the `coordinador` role only.

**Behavior**:
- Checks if user is authenticated
- Verifies user has `coordinador` role
- Returns 403 Forbidden if role check fails

**Error Message**: "No tienes permiso para acceder a esta página."

**Source**: RoleMiddleware.php:12-20
