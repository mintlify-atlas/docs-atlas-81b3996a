## Overview

The `PsicoController` manages all psychologist-related operations including viewing assigned students, creating psychological reports, managing referral details, and maintaining student history records.

**Namespace:** `App\Http\Controllers`

**File:** `app/Http/Controllers/PsicoController.php`

---

## Private Methods

### studentsByState()

Internal helper method to filter students by state for the authenticated psychologist.

```php
private function studentsByState(Request $request, string $state, string $label, string $routeName)
```

**Parameters:**
- `$request` - HTTP request with optional search query
- `$state` - State name to filter by
- `$label` - Display label for the view
- `$routeName` - Route name for pagination

**Logic:**
- Filters students by psychologist's assigned degrees
- Supports search by name, last name, document number, or full name
- Checks if students have psychoorientation annexes
- Returns paginated results (15 per page)

### store_report_student()

Stores a new psychological report and updates student status.

```php
public function store_report_student(Request $request, string $id)
```

<ParamField path="id" type="string" required>
  Student ID
</ParamField>

<ParamField body="number_documment" type="string" required>
  Document number (1-20 digits, unique)
</ParamField>

<ParamField body="name" type="string" required>
  Student's first name
</ParamField>

<ParamField body="last_name" type="string" required>
  Student's last name
</ParamField>

<ParamField body="degree" type="integer" required>
  Degree ID
</ParamField>

<ParamField body="group" type="integer" required>
  Group ID
</ParamField>

<ParamField body="age" type="integer" required>
  Student's age
</ParamField>

<ParamField body="state" type="integer" required>
  State ID (must exist in states table)
</ParamField>

<ParamField body="title_report" type="string" required>
  Report title
</ParamField>

<ParamField body="reason_inquiry" type="string" required>
  Reason for inquiry/investigation
</ParamField>

<ParamField body="recomendations" type="string" required>
  Recommendations
</ParamField>

<ParamField body="annex_one" type="file">
  PDF file (max 5MB)
</ParamField>

**File Storage:**
- PDF files stored in `public/annexes/student_{id}/`
- Filename format: `student_{id}_{YmdHis}.pdf`

**Business Logic:**

1. Updates student record (personal info and state)
2. Retrieves group director information
3. Uploads PDF annex if provided
4. Creates psychoorientation record with:
   - Psychologist ID (authenticated user)
   - Student information snapshot
   - Group director name
   - Report details
   - Annex file path

**Returns:** Redirects to remitted students list with success message

<Note>
  Uses database transactions to ensure data consistency. If any step fails, all changes are rolled back.
</Note>

## Database Tables Referenced

- `users_students` - Student records
- `referrals` - Referral information
- `psychoorientations` - Psychological reports
- `degrees` - Academic degrees
- `groups` - Student groups
- `states` - Student states
- `users_load_degrees` - Psychologist-degree assignments
- `users_teachers` - Teacher records

## File Storage

PDF annexes are stored using Laravel's `public` disk in the following structure:
```
storage/app/public/annexes/student_{id}/student_{id}_{timestamp}.pdf
```

## Authentication

All methods require psychologist authentication. Uses `Auth::id()` to identify the current psychologist.
