## Overview

The Academic Structure module provides the foundational organizational framework for the Bethlemitas Student Support System. It manages degrees (grade levels), groups (classes), and subject areas that teachers instruct.

## Teachers-Areas-Groups Relationship

The system uses a three-way pivot table (`teachers_areas_groups`) to track which teacher teaches which area in which group:

### Data Structure

```php
// Table: teachers_areas_groups
[
    'id_teacher' => 1,  // Teacher ID
    'id_area' => 2,     // Area (subject) ID
    'id_group' => 3,    // Group (class) ID
]
```

### Usage Example

When creating a teacher assignment:

```php
// Teacher 1 teaches Mathematics (Area 2) in Group A (3) and Group B (4)
DB::table('teachers_areas_groups')->insert([
    [
        'id_teacher' => 1,
        'id_area' => 2,  // Mathematics
        'id_group' => 3, // Group A
    ],
    [
        'id_teacher' => 1,
        'id_area' => 2,  // Mathematics
        'id_group' => 4, // Group B
    ]
]);
```

### Validation Logic

The system enforces business rules (`app/Http/Controllers/CreateController.php:165`):

```php
// Verify no area/group conflicts
foreach ($request->area_id as $area_id) {
    if (isset($request->groups_asig[$area_id])) {
        foreach ($request->groups_asig[$area_id] as $groupId) {
            $exists = Teachers_areas_group::where('id_area', $area_id)
                ->where('id_group', $groupId)
                ->exists();
            if ($exists) {
                $name_area = Area::find($area_id)->name_area;
                $name_group = Group::find($groupId)->group;
                return redirect()->back()->with('error', 
                    "Area {$name_area} is already taught in group {$name_group} by another teacher");
            }
        }
    }
}
```

<Note>
  This ensures that each subject in each class has only one assigned teacher, preventing scheduling conflicts.
</Note>

## Psychologist-Degree Assignments

Psychologists are assigned to specific grade levels through the `users_load_degrees` table:

### Data Structure

```php
// Table: users_load_degrees
[
    'id_user' => 5,     // Psychologist user ID
    'id_degree' => 1,   // Grade level ID
]
```

### Assignment Logic

When creating a psychologist (`app/Http/Controllers/CreateController.php:269`):

```php
// Verify no degree conflicts
foreach ($request->load_degree as $degree) {
    $exists = Users_load_degree::where('id_degree', $degree)->exists();
    if ($exists) {
        return redirect()->back()->with('error', 
            'Some grades are already assigned to another psychologist.');
    }
}

// Save grade assignments
foreach ($request->load_degree as $load_degree) {
    DB::table('users_load_degrees')->insert([
        'id_user' => $user->id,
        'id_degree' => $load_degree,
    ]);
}
```

<Warning>
  Each grade level can only be assigned to one psychologist at a time to ensure clear responsibility and prevent confusion.
</Warning>

## Sorting and Display

The system uses custom sorting for natural ordering:

```php
// Sort groups numerically then alphabetically
$groups = Group::orderByRaw('CAST(`group` AS UNSIGNED), `group`')->get();

// Sort degrees numerically then alphabetically
$degrees = Degree::orderByRaw('CAST(`degree` AS UNSIGNED), `degree`')->get();

// Sort areas alphabetically
$areas = Area::orderBy('name_area', 'asc')->get();
```

This ensures natural ordering like:
- Groups: 1, 2, 10, 11, A, B
- Degrees: 1, 2, 10, 11, Primero

## Data Integrity

The system maintains referential integrity through:

1. **Foreign Key Constraints**: Database-level constraints prevent orphaned records
2. **Application-Level Validation**: Business logic prevents invalid assignments
3. **Cascade Rules**: Deletions cascade appropriately or are prevented when dependencies exist
4. **Transaction Safety**: Multi-table operations use database transactions
