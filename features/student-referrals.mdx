## Overview

The Student Referral system allows teachers to submit students who need psychological support or special attention to the psycho-orientation team. Each referral includes detailed information about the student, the reason for referral, observations, and strategies already attempted.

    - Full name (first and last name)
    - Age (optional)
    - Degree and group
  </Step>
  
  <Step title="Provide Referral Details">
    Document the referral information:
    - **Reason for referral**: Primary concern requiring support
    - **Observations**: Detailed description of behaviors or situations
    - **Strategies**: What has already been tried
  </Step>
  
  <Step title="Submit Referral">
    The system validates and creates both the student record and the referral record in a database transaction.
  </Step>
</Steps>

<Note>
  When a referral is created, the system automatically:

  3. Sends an email notification to the assigned psychologist
</Note>

## State Validation

The system prevents duplicate referrals by checking the student's current state (`app/Http/Controllers/CreateReferralController.php:67`):

- **activo**: Student already referred and under review
- **en PIAR**: Student in PIAR process
- **en DUA**: Student in DUA process

```php
if ($state_info->state === 'activo') {
    return redirect()->back()->with(
        'info',
        'El estudiante ya fue remitido y está en revisión por el psicoorientador.'
    );
} elseif ($state_info->state === 'en PIAR') {
    return redirect()->back()->with(
        'info',
        'El estudiante ya está en proceso PIAR.'
    );
}
```

## Viewing Referrals

### Teacher View

Teachers can view their submitted referrals at `/index/students/remitted` (`app/Http/Controllers/CreateReferralController.php:130`):

```php
public function index_student_remitted(Request $request)
{
    $id_teacher = Auth::id();
    
    // Get students with 'activo' state sent by this teacher
    $query = Users_student::whereHas('states', function ($q) {
        $q->whereIn('state', ['activo']);
    })->where('sent_by', $id_teacher);
    
    // Apply search filter
    if ($request->filled('search')) {
        $searchTerm = $request->input('search');
        $query->where(function ($q) use ($searchTerm) {
            $q->where('name', 'LIKE', '%' . $searchTerm . '%')
                ->orWhere('last_name', 'LIKE', '%' . $searchTerm . '%')
                ->orWhere('number_documment', 'LIKE', '%' . $searchTerm . '%');
        });
    }
    
    $students = $query->with('latestReferral', 'group', 'degree')
        ->orderBy('name', 'asc')
        ->paginate(15);
}
```

## Email Notifications

When a referral is created, the system sends an automated email to the assigned psychologist using Laravel's queue system (`app/Http/Controllers/CreateReferralController.php:117`):

```php
Mail::to($psico_date->email)->queue(new CreatedReferralMail($user, $referral));
```

<Warning>
  The system requires that each grade has an assigned psychologist in the `users_load_degrees` table. If no psychologist is assigned, the referral cannot be created.
</Warning>
